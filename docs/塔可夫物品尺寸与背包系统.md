# 塔可夫风格物品尺寸与背包系统

> 参照《逃离塔科夫》的网格背包逻辑：不同物品占不同格子数，背包本身也有容量上限。
> 核心风险收益：高价值物品的体积决定你能不能带走它。

---

## 一、背包尺寸

### 1.1 背包分级

| 背包 | 网格 | 总格数 | 说明 |
|------|------|--------|------|
| 口袋 | 2×2 | 4 | 默认，只能装钥匙、弹药等 1×1 小件 |
| 小背包 | 4×3 | 12 | 初始搜刮所得，可装手枪+杂物 |
| 中型背包 | 5×4 | 20 | 最常见的背包，平衡容量与体积 |
| 大型背包 | 6×5 | 30 | 可装步枪+护甲+大量战利品 |
| 突击背包 | 6×7 | 42 | 最大容量，但目标大、搜刮慢 |

### 1.2 背包互套

大背包可以装小背包（占 2×2 格），但小背包里不能装更大的背包：

```javascript
function canFitInside(outer, inner) {
  // 检查 inner 是否能在 outer 里找到空间
  return inner.w <= outer.w && inner.h <= outer.h;
}
```

## 三、价值密度（核心决策指标）

价值密度 = 价值 ÷ 占用格数，决定某件物品是否值得占用背包空间。

| 物品 | 价值 | 格数 | 价值密度 | 优先级 |
|------|------|------|----------|--------|
| 遗物圣器 🔮 | 3000 | 4 | 750/格 | ⭐⭐⭐ 必拿 |
| 钻石 💎 | 600 | 1 | 600/格 | ⭐⭐⭐ |
| 宝石王冠 👑 | 2000 | 4 | 500/格 | ⭐⭐⭐ |
| 金佛像 🪷 | 1200 | 4 | 300/格 | ⭐⭐ |
| 金条 🪙 | 500 | 2 | 250/格 | ⭐⭐ |
| 翡翠手镯 💚 | 280 | 1 | 280/格 | ⭐⭐ |
| 古董怀表 ⏱️ | 320 | 2 | 160/格 | ⭐ |
| AWM | 1200 | 5 | 240/格 | ⭐⭐ |
| 破布 🧻 | 10 | 1 | 10/格 | ❌ 不拿 |

**塔可夫式决策**：背包空间有限，必须取舍。是拿 4 个钻石（4 格 × 600 = 2400）还是 1 个王冠（4 格 × 2000）？前者收益更高但需要找到 4 个。

## 四、胸挂与腰带（额外格子）

胸挂提供额外的固定尺寸网格，不占背包空间：

| 胸挂 | 网格 | 总格数 | 说明 |
|------|------|--------|------|
| 无 | 0×0 | 0 | — |
| 简易胸挂 | 2×2 | 4 | 仅放 1×1 小物 |
| 战术胸挂 | 2×3 | 6 | 可放弹匣/药包 |
| 突击胸挂 | 3×3 | 9 | 平衡容量 |
| 全能战术背心 | 4×3 | 12 | 最大胸挂容量 |

const RIGS = [

策略：胸挂放经常需要取用的物品（弹药、药包），背包放战利品。

## 五、总背包容量计算

```
总容量 = 胸挂网格 + 背包网格

示例：
  战术胸挂(2×3=6格) + 中型背包(5×4=20格) = 26格
  突击胸挂(3×3=9格) + 大型背包(6×5=30格) = 39格
```

```javascript
function getTotalCapacity(player) {
  const rig = player.rig || RIGS[0];
  const bp = player.backpack || BACKPACKS[0];
  return {
    rig:  { w: rig.w, h: rig.h, slots: rig.slots },
    backpack: { w: bp.w, h: bp.h, slots: bp.slots },
    total: rig.slots + bp.slots,
  };
}
```

| 找到 1 个王冠(4格)+3 个钻石(3格) | 中型背包(20格) | 轻松带走，继续搜 |
