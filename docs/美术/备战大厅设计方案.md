# 备战大厅（Lobby）设计方案

## 概述

参考《元气骑士》大厅：一个空旷的室内场景，散落着各种功能设施，玩家用 WASD 自由走动，走到设施附近出现交互提示，按 E 触发。

---

## 场景结构

大厅是一个**单房间的 BSP 地图**——没有走廊、没有敌人、四周墙壁围合。

```lua
local lobbyRoom = {
    x = 1, y = 1, w = 40, h = 30,
    cx = 20, cy = 15,
}
local lobbyRooms = { lobbyRoom }
```

用 IsoRenderer 渲染这个房间即可，和渲染地牢第一层一样。

## 交互流程

### 每帧检测

```lua
-- 在 Update 中
local nearestFacility = nil
local nearestDist = math.huge

for _, facility in ipairs(facilities) do
    -- 设施中心位置
    local fx = facility.gx + facility.w / 2
    local fy = facility.gy + facility.h / 2

    -- 玩家到设施中心的距离
    local dx = player.gx - fx
    local dy = player.gy - fy
    local dist = math.sqrt(dx * dx + dy * dy)

    if dist < facility.interactRange then
        if dist < nearestDist then
            nearestDist = dist
            nearestFacility = facility
        end
    end
end
```

### 交互提示

当 `nearestFacility` 不为空时：

1. 在设施正上方（屏幕坐标）显示 **"⋯"** 或 **"[E]"** 标记
2. 在屏幕底部显示提示文字：**"[E] 打开商店"**

```lua
-- 在 Draw 中，设施绘制之后
if nearestFacility then
    local fx = nearestFacility.gx + nearestFacility.w / 2
    local fy = nearestFacility.gy + nearestFacility.h / 2
    local sx, sy = IsoRenderer.gridToScreen(fx, fy)

    -- 设施上方的 E 标记
    nvg.FontSize(14)
    nvg.FontFace("sans")
    nvg.TextAlign("NVG_ALIGN_CENTER", "NVG_ALIGN_BOTTOM")
    nvg.FillColor(nvg.RGBA(255, 255, 255, 220))
    nvg.Text(sx, sy - 40, "[E]")

    -- 底部提示栏
    nvg.BeginPath()
    nvg.Rect(0, IsoRenderer.SH - 36, IsoRenderer.SW, 36)
    nvg.FillColor(nvg.RGBA(0, 0, 0, 180))
    nvg.Fill()
    nvg.FontSize(14)
    nvg.TextAlign("NVG_ALIGN_CENTER", "NVG_ALIGN_MIDDLE")
    nvg.FillColor(nvg.RGBA(255, 255, 255, 255))
    nvg.Text(IsoRenderer.SW / 2, IsoRenderer.SH - 18,
        "[E] " .. nearestFacility.name)
end
```

### 交互触发

```lua
-- 在 Input 处理中
if input.WasKeyPressed("E") and nearestFacility then
    nearestFacility.onInteract()
end
```

## 渲染管线

大厅的绘制顺序：

```
第 1 遍：地板（IsoRenderer.DrawDungeon → floor tiles）
第 2 遍：墙壁（IsoRenderer.DrawDungeon → wall tiles）
第 3 遍：设施精灵图（遍历 facilities，绘制 imagePath）
第 4 遍：NPC/其他角色
第 5 遍：玩家
第 6 遍：交互提示（nearestFacility 不为空时画 [E] 和底栏）
第 7 遍：光源（如果有烛台/吊灯等环境光）
```

设施在第 3 遍绘制，因为它们占据地面以上的空间，需要画在玩家前面或后面取决于位置。设施和玩家的前后关系按 depth 排序——设施中心 grid 的 depth 小于玩家则画在前面，大于则画在后面。简化处理：所有设施画在玩家之前（玩家始终在最前面）。

## 需要 MCP 生图的素材

大厅的设施需要等距视角的精灵图。一次生成一套：

| 设施 | 生图 prompt 要点 |
|------|-----------------|
| 角色选择台 | "isometric view, pixel art, wooden table with crown symbol on top, game lobby furniture, 64x48" |
| 商店柜台 | "isometric pixel art shop counter, wooden, with items displayed on top, 64x40" |
| 仓库箱子 | "isometric pixel art large wooden crate, metal bands, game stash, 48x48" |
| 技能树 | "isometric pixel art ancient stone monument, glowing runes on surface, magical skill tree, game lobby, 64x56" |
| 传送门 | "isometric pixel art blue glowing portal, magical swirl, game teleport, 64x64" |
| 工作台 | "isometric pixel art workbench, tools on table, anvil, game crafting, 64x48" |
| 邮箱 | "isometric pixel art mailbox, red, game daily reward, 32x40" |

png 透明背景，尺寸按上述生成后直接使用。

## 主循环集成

```lua
-- 场景切换
local SCENE = "lobby"  -- "lobby" | "dungeon"

function Update(timeStep)
    if SCENE == "lobby" then
        Player.Update(timeStep)
        CheckFacilityProximity()  -- 更新 nearestFacility
        if input.WasKeyPressed("E") and nearestFacility then
            nearestFacility.onInteract()
        end
    elseif SCENE == "dungeon" then
        -- 地牢逻辑...
    end
end

function Draw(nvg)
    if SCENE == "lobby" then
        IsoRenderer.DrawDungeon(nvg, lobbyCells, lobbyRooms, ...)
        DrawFacilities(nvg)         -- 第 3 遍
        DrawNPCs(nvg)               -- 第 4 遍
        Player.Draw(nvg)            -- 第 5 遍
        DrawInteractionPrompt(nvg)  -- 第 6 遍
        DrawLighting(nvg, lobbyLights)  -- 第 7 遍
    elseif SCENE == "dungeon" then
        -- ...
    end
end
```

## 一句话给 AI / 引擎

> "单房间 BSP 场景，40×30 grid，等距渲染。布置 N 个设施（角色选择/商店/仓库/技能树/传送门/工作台/邮箱），每个设施占 2~4 格，有精灵图和交互回调。玩家 WASD 移动，每帧检测到最近设施的距离，进入 interactRange（2~2.5 格）时在设施上方显示 [E] 标记、底部显示提示文字，按 E 触发交互打开对应 UI 界面。设施用 MCP 生成的等距视角 PNG 精灵绘制。"
