# 修勾大逃亡 — 光照与后期效果系统

> 实现《挺进地牢》风格的实时光照、暗角、动态粒子发光
> 适配语言：Lua 5.4
> 适配引擎：UrhoX（NanoVG 2D 渲染 + Yoga UI）

---

## 一、架构总览

### 1.1 渲染管线

```
每帧渲染顺序（由外到内）：

1. 绘制地牢场景（地板 → 墙壁 → 掩体 → 敌人 → 玩家 → 粒子/弹壳）
2. 生成光照图（Light Map）并叠加
3. 叠加暗角（Vignette）
4. 绘制 UI 层（小地图、HUD）
```

### 1.2 光照图（Light Map）概念

光照图是一个与屏幕等大的半透明图层，记录每个像素的光照强度。

- 使用 **NanoVG 径向渐变**绘制每个光源
- 光源形状：从光源中心（亮）到边缘（完全透明）的渐变圆
- 所有光源绘制到同一张图上后，通过 **Multiply 混合模式**叠加到主场景上

### 1.3 光源数据结构

```lua
--- @class LightSource
--- @field x number        世界坐标 X
--- @field y number        世界坐标 Y
--- @field radius number   光照半径（像素）
--- @field r number        颜色 R (0~1)
--- @field g number        颜色 G (0~1)
--- @field b number        颜色 B (0~1)
--- @field intensity number 强度倍率 (0~1)
--- @field life number     剩余帧数（-1 表示永久光源）
```

## 二、P0 — 实时光照系统

### 2.1 环境光基准

整个画面的基础亮度由环境光决定。没有光源照射的区域不会完全漆黑——保留最低可见度。

```lua
-- 全局光照参数
local AmbientColor = { r = 0.1, g = 0.08, b = 0.12 }  -- 深紫灰环境光
```

### 2.2 光源管理器

```lua
--- 光照管理器
local LightManager = {
    staticLights = {},   -- 固定光源（火炬、环境点）
    dynamicLights = {},  -- 动态光源（枪火、爆炸、掉落物）
}

--- 注册一个静态光源
--- @param x number
--- @param y number
--- @param radius number
--- @param r number
--- @param g number
--- @param b number
--- @param intensity number
function LightManager:addStatic(x, y, radius, r, g, b, intensity)
    table.insert(self.staticLights, {
        x = x, y = y,
        radius = radius,
        r = r, g = g, b = b,
        intensity = intensity or 1.0,
        life = -1,  -- -1 = 永久
    })
end

--- 添加一个动态光源（枪火/爆炸）
--- @param x number
--- @param y number
--- @param radius number
--- @param r number
--- @param g number
--- @param b number
--- @param lifeFrames number  持续帧数
--- @param intensity number
function LightManager:addDynamic(x, y, radius, r, g, b, lifeFrames, intensity)
    table.insert(self.dynamicLights, {
        x = x, y = y,
        radius = radius,
        r = r, g = g, b = b,
        life = lifeFrames,
        intensity = intensity or 1.0,
    })
end

--- 每帧更新（移除过期的动态光源）
function LightManager:update()
    local alive = {}
    for _, light in ipairs(self.dynamicLights) do
        if light.life > 0 then
            light.life = light.life - 1
            table.insert(alive, light)
        elseif light.life == -1 then
            table.insert(alive, light)
        end
        -- life = 0 时丢弃
    end
    self.dynamicLights = alive
end

--- 收集当前帧所有光源
function LightManager:getAllLights()
    local all = {}
    for _, l in ipairs(self.staticLights) do
        table.insert(all, l)
    end
    for _, l in ipairs(self.dynamicLights) do
        table.insert(all, l)
    end
    return all
end
```

### 2.3 渲染光照图

```lua
--- 在 NanoVG 中渲染光照图
--- @param vg userdata   NanoVG context (UrhoX 传入)
--- @param lights table  光源列表
--- @param screenW number 屏幕宽度
--- @param screenH number 屏幕高度
--- @param cameraX number 摄像机 X 偏移
--- @param cameraY number 摄像机 Y 偏移
function renderLightMap(vg, lights, screenW, screenH, cameraX, cameraY)
    -- 步骤 1：保存当前 NanoVG 状态
    nvgSave(vg)

    -- 步骤 2：绘制环境阴影底色
    -- 以 Multiply 模式叠加到场景上，所以底色越深场景越暗
    nvgBeginPath(vg)
    nvgRect(vg, 0, 0, screenW, screenH)
    nvgFillColor(vg, nvgRGBA(
        math.floor(AmbientColor.r * 255),
        math.floor(AmbientColor.g * 255),
        math.floor(AmbientColor.b * 255),
        200  -- Alpha 控制整体暗度
    ))
    nvgFill(vg)

    -- 步骤 3：对每个光源，在对应屏幕位置画径向渐变
    for _, light in ipairs(lights) do
        -- 世界坐标转屏幕坐标
        local sx = light.x - cameraX
        local sy = light.y - cameraY

        -- 不在屏幕范围内的光源跳过
        if sx < -light.radius or sx > screenW + light.radius
        or sy < -light.radius or sy > screenH + light.radius then
            goto continue
        end

        -- 创建径向渐变
        -- NanoVG 的径向渐变：中心+(内半径) 到 边缘+(外半径)
        local grad = nvgRadialGradient(vg,
            sx, sy, 0,                              -- 中心点，内半径=0
            sx, sy, light.radius * light.intensity,  -- 外圈
            nvgRGBA(
                math.floor(light.r * 255),
                math.floor(light.g * 255),
                math.floor(light.b * 255),
                180  -- 中心完全不透明（擦亮区域）
            ),
            nvgRGBA(0, 0, 0, 0)  -- 边缘完全透明
        )

        -- 用 'lighter' 混合模式叠加光源
        -- NanoVG 使用 globalCompositeOperation 控制混合
        -- 注意：NanoVG 原生可能不支持 lighter 模式
        -- 替代方案见下方 2.4 节
        nvgBeginPath(vg)
        nvgRect(vg, sx - light.radius, sy - light.radius,
                light.radius * 2, light.radius * 2)
        nvgFillPaint(vg, grad)
        nvgFill(vg)

        ::continue::
    end

    -- 步骤 4：将光照图以 Multiply 模式叠加到主场景
    -- 实际实现中，这一步通过 UrhoX 的渲染管线完成
    -- 见下方 2.5 节

    nvgRestore(vg)
end
```

### 2.4 混合模式说明

NanoVG 原生不直接支持 `multiply` 或 `lighter` 混合模式。在 UrhoX 中有两种实现路径：

**路径 A：在 UrhoX 材质系统层处理**

```
1. 将 NanoVG 场景渲染到一张 RenderTexture（RT）
2. 将光照图渲染到另一张 RT
3. 在 UrhoX 中创建一个全屏 Quad 材质，
   使用 multiply 混合模式将两张 RT 合成
```

**路径 B：用 NanoVG 的 GlobalAlpha 模拟**

```
1. 直接在主场景绘制完毕后，
   用半透明黑色径向渐变"刮亮"出光照区域
2. 不是真正的光照，但效果接近
3. 性能开销最低
```

```lua
--- 路径 B 的简化实现
--- 用半透明黑色覆盖全屏，再用光源"刮掉"黑色
function renderLightMap_simple(vg, lights, screenW, screenH, camX, camY)
    -- 1. 覆盖半透明黑色
    nvgBeginPath(vg)
    nvgRect(vg, 0, 0, screenW, screenH)
    nvgFillColor(vg, nvgRGBA(20, 18, 28, 160))  -- 暗色半透明覆盖层
    nvgFill(vg)

    -- 2. 用光源"挖掉"黑色（画透明径向渐变）
    for _, light in ipairs(lights) do
        local sx = light.x - camX
        local sy = light.y - camY
        if sx < -light.radius or sx > screenW + light.radius
        or sy < -light.radius or sy > screenH + light.radius then
            goto continue
        end

        local grad = nvgRadialGradient(vg,
            sx, sy, 0,
            sx, sy, light.radius * light.intensity,
            nvgRGBA(0, 0, 0, 0),          -- 中心：完全透明（全亮）
            nvgRGBA(0, 0, 0, 200)          -- 边缘：半透明黑（过渡到暗）
        )
        nvgBeginPath(vg)
        nvgRect(vg, sx - light.radius, sy - light.radius,
                light.radius * 2, light.radius * 2)
        nvgFillPaint(vg, grad)
        nvgFill(vg)

        ::continue::
    end
end
```

**路径 B 的效果对照：**

| 位置 | 有光源 | 无光源 |
|------|--------|--------|
| 中心 | 原始画面（不受影响） | 半透明黑覆盖（变暗但可见） |
| 边缘 | 渐暗过渡 | 完全暗色 |

### 2.5 UrhoX 渲染管线集成（推荐路径 A）


## 三、P1 — 暗角（Vignette）

### 3.1 原理

在画面四个角叠加一个径向渐变，中心透明、边缘半透明黑色。让玩家的注意力集中在画面中央。

### 3.2 实现

```lua
--- 渲染暗角
--- @param vg userdata   NanoVG context
--- @param screenW number
--- @param screenH number
--- @param intensity number  0~1，暗角强度（推荐 0.4）
--- @param radius number     暗角半径比例（推荐 0.6，中心 60% 区域不受影响）
function renderVignette(vg, screenW, screenH, intensity, radius)
    if intensity <= 0 then return end

    nvgSave(vg)

    -- 创建一个从屏幕中心到边缘的径向渐变
    local cx = screenW / 2
    local cy = screenH / 2
    local maxDist = math.sqrt(cx * cx + cy * cy)  -- 对角半长

    local grad = nvgRadialGradient(vg,
        cx, cy, maxDist * radius,          -- 内圈：radius 比例内完全透明
        cx, cy, maxDist,                    -- 外圈：屏幕角落
        nvgRGBA(0, 0, 0, 0),               -- 内圈：完全透明
        nvgRGBA(0, 0, 0, math.floor(255 * intensity))  -- 外圈：半透明黑
    )

    -- 覆盖全屏
    nvgBeginPath(vg)
    nvgRect(vg, 0, 0, screenW, screenH)
    nvgFillPaint(vg, grad)
    nvgFill(vg)

    nvgRestore(vg)
end
```

### 3.3 层段暗角强度

| 层段 | 强度 | 说明 |
|------|------|------|
| 1~4 | 0.25 | 浅暗角，新手指引期不压视野 |
| 6~9 | 0.35 | 标准暗角 |
| 11~14 | 0.40 | 偏重，配合熔岩主题 |
| 16~19 | 0.50 | 重暗角，配合虚空禁区的压迫感 |

## 四、P4 — 动态粒子发光

### 4.1 原理

P4 本质上是 P0 的子集——往光照图里添加动态的、短寿命的光源。不需要独立的渲染层，只需要在每帧往 LightManager 里注册临时光源。

### 4.2 枪火光

```lua
--- 玩家或敌人开枪时触发
--- @param x number 枪口位置 X
--- @param y number 枪口位置 Y
--- @param weapon string 武器类型
function onGunFire(x, y, weapon)
    -- 定义每种武器的枪火参数
    local gunfirePresets = {
        pistol    = { radius = 60,  r = 1.0, g = 0.9, b = 0.5, life = 3,  intensity = 1.2 },
        smg       = { radius = 50,  r = 1.0, g = 0.85, b = 0.4, life = 2,  intensity = 1.0 },
        rifle     = { radius = 80,  r = 1.0, g = 0.8, b = 0.3, life = 4,  intensity = 1.5 },
        shotgun   = { radius = 100, r = 1.0, g = 0.7, b = 0.2, life = 5,  intensity = 1.8 },
        sniper    = { radius = 120, r = 1.0, g = 0.9, b = 0.6, life = 6,  intensity = 2.0 },
        smg       = { radius = 45,  r = 1.0, g = 0.85,b = 0.4, life = 2,  intensity = 0.9 },
    }

    local preset = gunfirePresets[weapon] or gunfirePresets.pistol

    -- 主枪火光
    LightManager:addDynamic(
        x, y,
        preset.radius,
        preset.r, preset.g, preset.b,
        preset.life,
        preset.intensity
    )

    -- 附加：枪口周围的小火星（不发光，只做粒子效果）
    -- 使用 NanoVG 绘制，不参与光照
    spawnMuzzleParticles(x, y, 3, weapon)
end
```

### 4.3 爆炸光

```lua
--- 手雷/火箭/炸药桶爆炸
--- @param x number
--- @param y number
function onExplosion(x, y, tier)
    local tierData = {
        small  = { radius = 150, life = 12, intensity = 2.0 },
        medium = { radius = 200, life = 18, intensity = 2.5 },
        large  = { radius = 280, life = 24, intensity = 3.0 },
    }

    local data = tierData[tier] or tierData.medium

    -- 爆炸闪光
    LightManager:addDynamic(x, y, data.radius, 1.0, 0.6, 0.1, data.life, data.intensity)

    -- 爆炸余烬（缓慢衰减的小光点）
    -- 在爆炸中心周围随机生成 6~10 个小动态光源
    for i = 1, 8 do
        local angle = math.random() * math.pi * 2
        local dist = math.random() * data.radius * 0.6
        LightManager:addDynamic(
            x + math.cos(angle) * dist,
            y + math.sin(angle) * dist,
            30 + math.random() * 40,        -- 小半径
            1.0, 0.5 + math.random() * 0.3, 0.1,
            6 + math.random() * 8,          -- 比主闪光多持续几帧
            0.5 + math.random() * 0.5
        )
    end
end
```

### 4.5 光源衰减优化

当动态光源数量超过性能阈值时，优先淘汰视野外的光源和低强度光源。

```lua
--- 每帧更新后的性能保护
--- @param maxLights number 每帧最大动态光源数（推荐 20~30）
function LightManager:trim(maxLights)
    if #self.dynamicLights <= maxLights then return end

    -- 按剩余生命排序，淘汰即将消失的
    table.sort(self.dynamicLights, function(a, b)
        return a.life > b.life
    end)

    -- 保留前 maxLights 个
    local trimmed = {}
    for i = 1, math.min(maxLights, #self.dynamicLights) do
        table.insert(trimmed, self.dynamicLights[i])
    end
    self.dynamicLights = trimmed
end
```

## 五、渲染管线完整调用顺序

```lua
--- 主渲染函数（每帧调用）
--- @param vg userdata   NanoVG context (UrhoX 传入)
function onRenderFrame(vg, dt)
    -- ========== 更新逻辑 ==========
    LightManager:update()
    LightManager:trim(30)  -- 最多保留 30 个动态光源

    -- ========== 1. 绘制主场景 ==========
    drawFloor(vg)          -- 地板
    drawWalls(vg)          -- 墙壁
    drawCovers(vg)         -- 掩体
    drawEnemies(vg)        -- 敌人
    drawPlayer(vg)         -- 玩家
    drawParticles(vg)      -- 子弹、弹壳、粒子

    -- ========== 2. P0：光照 ==========
    local allLights = LightManager:getAllLights()
    local screenW, screenH = getScreenSize()
    local camX, camY = getCameraPosition()

    -- 使用路径 B（简化版，不依赖 RenderTarget）
    renderLightMap_simple(vg, allLights, screenW, screenH, camX, camY)

    -- ========== 3. P1：暗角 ==========
    local floor = getCurrentFloor()
    local vignetteIntensity = getVignetteForFloor(floor)
    renderVignette(vg, screenW, screenH, vignetteIntensity, 0.6)

    -- ========== 4. 绘制 UI ==========
    drawHUD(vg)            -- 血条、弹药、背包
    drawMiniMap(vg)        -- 小地图（不受光照影响）
end
```

## 七、性能参考

| 效果 | 光源数 | 帧耗时（估算） | 备注 |
|------|--------|--------------|------|
| 仅 P0（路径 B） | 10 个静态 | < 0.5ms | 火炬 + 环境 |
| P0 + 10 个动态 | 20 个 | ~1ms | 战斗场景 |
| P0 + P1 | 同上有无 | +0.1ms | 暗角几乎免费 |
| P0 + P1 + P4 满负荷 | 30+ 个 | ~2ms | 爆炸+连射极限情况 |

路径 A（RenderTarget 合成）比路径 B 重约 3~5 倍，但效果更准确。建议先用路径 B 开发，性能有余量再升级到路径 A。
